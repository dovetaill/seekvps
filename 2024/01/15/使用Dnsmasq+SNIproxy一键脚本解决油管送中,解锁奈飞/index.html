一、何谓送中机
送中机，是指机器IP被Google判定为中国大陆的俗称，也称：宋仲基。

通常情况下，如果低价机，或者机场多的ISP，很容易开出送中机。

1、送中机的问题：
Youtube Premium 不能用 (YouTube Premium 和 YouTube Music Premium 不支持中国)
Google play不能下载软件
Youtube 不支持后台播放
搜索结果也可能不一样 (当然, 可用通过用 google.com/ncr 去搜索)

2、送中的原因：
通常情况下，浏览器或APP开了GPS定位(比如手机/电脑)权限。使用VPS的IP访问谷歌服务时, Google可能把GPS定位和IP关联起来, 导致IP被标记为中国。

3、如何检测是否送中：
只需要运行如下多媒体解锁检测脚本即可：

bash <(curl -L -s https://raw.githubusercontent.com/lmc999/RegionRestrictionCheck/main/check.sh)
如图即送中

4、常规解决送中的办法：
关闭浏览器或手机APP的定位(GPS)权限
电脑移除定位权限: Chrome – 设置 – 隐私设置和安全性, 网站设置 – 位置信息, 移除谷歌的相关站点.

手机关闭APP的定位权限: 应用的权限管理, 定位服務/位置信息 的权限.

去Google的帮助中心, 报告 IP 问题
一般提交后, 可能1周以上才会有效果(当然也不一定被采纳), 可以间隔1周提交一次.

电脑端的谷歌浏览器, 通过Location Guard插件, 强制标记GPS定位.
安装后, Fixed Location 菜单, 标记需要模拟GPS定位的位置, 根据自己VPS IP, 在地图上找到对应的国家/城市。

Options 菜单, Default level: 改成 Use fixed location 。

经常使用Google搜索/油管等服务, 长期给谷歌推送定位。

但是，即便用这种方式，IP恢复也很困难，即便能恢复到机器所在国，但仍然需要时间。

有办法马上就解决送中问题么？

有的。

二、使用Dnsmasq+SNIproxy解决送中

1、概述
原理：使用Dnsmasq的DNS将网站解析劫持到SNI proxy反向代理的页面上。

借鉴原作者myxuchangbin在Github上的一键脚本，我修改了部分代码。

https://github.com/codionx/dnsmasq_sniproxy_install

原作者的脚本主要用来解决如Netflix解锁的问题，而实际上，劫持方式解锁Netflix，和劫持方式突破送中，是同一原理，无非是把DNS的分流规则修改成Youtube就好。

2、技术逻辑
在不送中的机器上运行Dnsmasq+SNIproxy服务；
设置不送中机器的IP为送中机器的DNS；
即可

3、安装
在不送中机上运行

wget –no-check-certificate -O dnsmasq_sniproxy_youtube.sh https://raw.githubusercontent.com/codionx/dnsmasq_sniproxy_install/master/dnsmasq_sniproxy_youtube.sh && bash dnsmasq_sniproxy_youtube.sh -f
4、修改DNS
在送中机上

nano /etc/resolv.conf
删除全部内容，并添加

nameserver 不送中机IP
5、检测
使用多媒体解锁检测脚本即可

bash <(curl -L -s https://raw.githubusercontent.com/lmc999/RegionRestrictionCheck/main/check.sh)
如图即正常

6、卸载
wget –no-check-certificate -O dnsmasq_sniproxy_youtube.sh https://raw.githubusercontent.com/codionx/dnsmasq_sniproxy_install/master/dnsmasq_sniproxy_youtube.sh && bash dnsmasq_sniproxy_youtube.sh -u
至此，已经基本解决送中问题。

但，如果你恰好碰上NF、D+解锁机送中，又要怎么处理呢？

三、解决NF解锁机送中问题

1、只解决NF解锁机送中问题
这个比较简单，只需要把不送中机的IP设定为NF解锁机的DNS即可。

2、同时解决NF解锁机送中及NF机给其他机解锁的问题
这个标题有点绕，实际上就是，

解决NF解锁机的Youtube访问
在NF解锁机上跑Dnsmasq+SNIproxy脚本，并作为其他不解锁NF且送中的机的DNS使用

（1）在NF解锁机上运行完整版脚本
wget –no-check-certificate -O dnsmasq_sniproxy_all.sh https://raw.githubusercontent.com/codionx/dnsmasq_sniproxy_install/master/dnsmasq_sniproxy_all.sh && bash dnsmasq_sniproxy_all.sh -f
